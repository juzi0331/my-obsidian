# 🐍 Python 进阶：函数传参方式

> [!ABSTRACT]  Python 进阶：函数传参方式
传参方式是指在调用函数时，将**实参**（实际数据）传递给**形参**（定义时的变量）的不同方式。

### 1. 位置参数 (Positional Arguments)

这是最基础的传参方式。调用函数时，根据参数在定义时的**位置顺序**依次传入。

- **核心要求**：调用时的参数顺序必须与函数定义时的顺序**完全一致**。
- **代码示例**：
```Python
# 按照 name, age, gender, city 的顺序传入
stu = reg_stu("橘子", "18", "男", "青岛")
```

### 2. 关键字参数 (Keyword Arguments)

调用函数时，通过 `形参名 = 值` 的形式来明确指定数据给哪个参数。

- **核心优势**：参数的**顺序不限**，提高了代码的可读性。
- **代码示例**：
```Python
# 即使改变顺序也可以准确对应
stu1 = reg_stu(name = "橘子" , age = "19", gender = "男" , city = "济南")
```

### 3. 混合传参 (Mixed Arguments)

在同一个函数调用中，可以同时使用位置参数和关键字参数。

- **核心要求**：**关键字参数必须写在位置参数之后**。
- **代码示例**：
```Python
# 前两个使用位置传参，后两个使用关键字传参
stu2 = reg_stu("橘子", "20" , gender ="男", city = "上海")
```

| **传参方式**  | **语法示例**         | **顺序要求**   | **备注**        |
| --------- | ---------------- | ---------- | ------------- |
| **位置传参**  | `func(a, b)`     | **严格**一致   | 简单直观，但参数多时易混淆 |
| **关键字传参** | `func(b=2, a=1)` | **无**要求    | 更加清晰，避免传错参数   |
| **混合传参**  | `func(1, b=2)`   | 位置参数**在前** | 必须遵循位置参数优先原则  |
### 💡 核心要点

1. **实参概念**：在调用函数时传递的真实数据被称为“实参”。
2. **避免歧义**：关键字参数之间没有顺序要求，但一旦开始使用关键字传参，后续的所有参数都必须是关键字形式。
3. **案例演示**：在你提供的 `reg_stu` 函数中，无论哪种传参方式，最终都能正确返回包含学生信息的字典。

**优化说明：**

- **规则强调**：特别突出了混合传参时“位置参数在前”的强制性要求。
- **结构化对比**：使用表格清晰呈现了三种方式的差异。
- **代码关联**：将理论与你代码中的 `stu`、`stu1`、`stu2` 变量一一对应，方便复习。
