# 🐍 Python 基础：函数 (Function)


> [!ABSTRACT] 🐍 Python 基础：函数
> 函数是组织好的、可重复使用的、用来实现**特定功能**的代码片段。
> - **核心价值**：提高代码的复用性，> 减少重复劳动，使程序结构更清晰。


### 1. 函数的语法结构

在 Python 中，使用 `def` 关键字来声明函数。

```Python
# 定义函数
def 函数名(参数列表):
    """文档注释 (可选)"""
    函数体 (实现功能的逻辑代码)
    return 返回值

# 调用函数
函数名(参数)
```

### 2. 函数的核心规则

根据你的练习总结，函数在使用时必须遵循以下原则：

| **规则**       | **描述**                              |
| ------------ | ----------------------------------- |
| **先定义后调用**   | 函数必须在被调用之前出现在代码中，否则会报“未定义”错误。       |
| **定义时不执行**   | 在使用 `def` 定义函数时，内部的代码块并不会执行。        |
| **调用才触发**    | 只有当函数被显式调用（通过 `函数名()`）时，函数体的逻辑才会运行。 |
| **参数/返回值可选** | 根据需求，参数列表和 `return` 语句都是可有可无的。      |

### 3. 实战代码演示

以下是基于你提供的习题整理的规范代码：

```Python
# 1. 定义阶段：此时程序只是记住了这个函数名，并不会打印任何内容
def out_line(): 
    """打印分隔线"""
    print('------------')

# 2. 调用阶段：此时才会执行函数体内的 print 语句
out_line()
```

### 💡 核心要点总结

1. **代码片段化**：将经常需要执行的操作（如打印特定的分隔符 `------------`）封装进函数，可以极大地简化主程序的编写。
2. **缩进要求**：函数体内的代码必须相对于 `def` 语句保持一致的缩进。
3. **命名建议**：函数名应体现其功能（如 `out_line`），使用小写字母及下划线命名法（snake_case）。

**优化说明：**

- **清晰的生命周期说明**：明确区分了“定义阶段”和“调用阶段”的不同行为。
- **参数灵活性提示**：强调了参数和返回值的可选性，这对于初学者理解不同复杂度的函数非常有帮助。
- **规范化代码**：将 `out_xx` 优化为 `out_line`，使其命名更具语义化。
