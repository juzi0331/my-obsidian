# 🐍 Python 进阶：函数默认参数 (Default Arguments)

### 1. 什么是默认参数？

默认参数（也称为**缺省参数**）是在定义函数时，提前为某些形参预设一个值。

- **核心作用**：在调用函数时，如果不为这些参数传递具体数值，Python 将自动使用预设的默认值。
- **代码价值**：简化了高频场景下的函数调用，减少了重复传参的麻烦。

### 2. 定义规则与限制

在定义带默认值的参数时，必须严格遵守 Python 的语法要求：

**关键准则：尾部原则** **默认参数必须放在没有默认值的参数列表的后面**。

- ✅ 正确：`def func(a, b=10):`
- ❌ 错误：`def func(a=10, b):`

### 3. 实战案例：学生注册系统

通过设置 `city='北京'` 作为默认值，我们可以看到不同调用方式下的灵活性：

```Python
# 定义函数：设置 city 的缺省值为 '北京'
def reg_stu(name, age, gender, city='北京'):
    print(f"注册成功,姓名:{name},年龄:{age},性别:{gender},城市:{city}")
    return {"name": name, "age": age, "gender": gender, "city": city}

# 情况 A：使用默认值
# 调用时不传 city 参数，系统自动补全为 '北京'
stu1 = reg_stu("张三", 18, "男")
print(stu1)  # 输出结果包含 'city': '北京'

# 情况 B：覆盖默认值
# 调用时传入新的城市 '深圳'，默认值 '北京' 会被替换
stu2 = reg_stu("赵四", 22, "男", "深圳")
print(stu2)  # 输出结果包含 'city': '深圳'
```
### 4.默认参数对比总结

| **场景**      | **是否传参** | **执行结果**                  |
| ----------- | -------- | ------------------------- |
| **高频通用场景**  | ❌ 否      | 自动使用定义时预设的**默认值**。        |
| **特殊个性化场景** | ✅ 是      | 以调用时传入的**实际数据**为准（覆盖默认值）。 |
### 💡 核心注意点

1. **参数顺序**：在定义阶段，一定要把必填参数写在前面，可选参数（带默认值的）写在最后面，否则程序会报错。
2. **不可变对象**：在进阶开发中，建议默认参数尽量使用**不可变类型**（如字符串、数字、元组）。如果使用列表等可变对象作为默认值，可能会产生意想不到的“记忆效应”。
3. **灵活组合**：默认参数可以与我们之前学的“关键字参数”配合使用，实现跳过部分默认参数进行传参。
